## ШАГ 1: Зарегистрируйся в Яндекс.Облаке

1. Зайди на https://console.yandex.cloud
2. Нажми «Войти» → войди через Яндекс.Почту
3. Создай новую организацию (если нет)
4. Создай новый **Billing account** (платёжный аккаунт)
   - Выбери «Физическое лицо»
   - Привяжи карту — Яндекс.Облако даст **1000₽ приветственного кредита**
   - **Free tier**: бесплатная виртуальная машина 2vCPU / 4GB RAM / 30GB диск навсегда

---

## ШАГ 2: Создай виртуальную машину

1. В консоли Яндекс.Облака перейди в **Compute Cloud → Виртуальные машины**
2. Нажми **Создать ВМ**
3. Настройки:
   - **Имя**: `nastolki-server`
   - **Зона доступности**: `ru-central1-a`
   - **Образ**: Ubuntu 22.04 LTS
   - **Конфигурация**: `standard-v3 · 2 vCPU · 4 GB RAM` (подходит под free tier)
   - **Диск**: SSD 30 GB
   - **Сетевые настройки**: Включить **публичный IP-адрес** (статический)
   - **SSH-ключ**: создай новый или вставь свой публичный ключ
4. Нажми **Создать ВМ**
5. Запомни **публичный IP-адрес** сервера (например `84.201.xxx.xxx`)

## ШАГ 7: Запускай сервер постоянно (PM2)

```bash
# Устанавливаем PM2 — менеджер процессов
sudo npm install -g pm2

# Запускаем сервер через PM2
cd ~/nastolki/backend
pm2 start server.js --name nastolki-backend

# Автозапуск при перезагрузке
pm2 startup
pm2 save

# Проверяем
pm2 status  # должно быть: nastolki-backend | online
pm2 logs nastolki-backend  # смотреть логи
```

## ШАГ 8: Настрой Nginx (веб-сервер для сайта)

```bash
# Устанавливаем Nginx
sudo apt install -y nginx

# Создаём конфиг сайта
sudo nano /etc/nginx/sites-available/nastolki
```

Вставь этот конфиг:
```nginx
server {
    listen 80;
    server_name 84.201.xxx.xxx;  # замени на свой IP или домен

    # Фронтенд — главный сайт
    location / {
        root /home/ubuntu/nastolki/frontend;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # Админка
    location /admin {
        root /home/ubuntu/nastolki;
        try_files /admin/admin.html =404;
    }

    # Бэкенд API
    location /api/ {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # Здоровье сервера
    location /health {
        proxy_pass http://localhost:3000;
    }
}
```

```bash
# Включаем сайт
sudo ln -s /etc/nginx/sites-available/nastolki /etc/nginx/sites-enabled/
sudo nginx -t  # проверка конфига
sudo systemctl restart nginx
sudo systemctl enable nginx

# Разрешаем порт 80 в файрволе
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw allow 22/tcp
sudo ufw enable
```

## ШАГ 9: Обнови API_URL в коде сайта

Открой `frontend/index.html` на строке:
```javascript
const API_URL = window.location.hostname === 'localhost' 
  ? 'http://localhost:3000' 
  : 'https://ВАШ_СЕРВЕР.yandexcloud.net'; // <-- ЗАМЕНИТЕ
```

Замени на:
```javascript
const API_URL = 'http://84.201.xxx.xxx';  // твой IP
```

Аналогично в `admin/admin.html`:
```javascript
const API = '...' ? '...' : 'http://YOUR_SERVER_IP:3000';
```
Замени `YOUR_SERVER_IP` на твой IP.

Потом загрузи обновлённые файлы на сервер снова.

## ОПЦИОНАЛЬНО: Уведомления в Telegram

Чтобы получать уведомления о новых заявках:

1. Напиши боту @BotFather в Telegram: `/newbot`
2. Следуй инструкциям, получи **токен бота** (типа `1234567890:AAH...`)
3. Напиши своему боту первое сообщение
4. Зайди на: `https://api.telegram.org/bot<ТОН>/getUpdates`
5. Найди в ответе `"chat":{"id":XXXXXXX}` — это твой **chat_id**
6. Вставь оба значения в `.env` файл на сервере
7. Перезапусти: `pm2 restart nastolki-backend`

Теперь при каждой новой заявке ты получишь сообщение в Telegram!

## ОПЦИОНАЛЬНО: Домен и HTTPS

Если хочешь красивый домен (например nastolki-ryadom.ru):
1. Купи домен на reg.ru или nic.ru (~150-300₽/год)
2. Настрой A-запись DNS → твой IP сервера
3. Установи SSL сертификат:
```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d nastolki-ryadom.ru
```

Это даст тебе **HTTPS** — сайт будет работать по `https://nastolki-ryadom.ru`
